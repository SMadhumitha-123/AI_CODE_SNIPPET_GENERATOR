Here are three simple, commonly used ways to reverse a string in Python, each with a short explanation and example.

1) Using slicing (most idiomatic and concise)
```python
def reverse_string_slicing(s: str) -> str:
    # s[::-1] returns a new string with characters stepped backwards
    return s[::-1]

# Example
text = "Hello, world!"
print(reverse_string_slicing(text))  # !dlrow ,olleH
```
Explanation: s[::-1] uses Python slicing with a step of -1 to produce the reversed string. Time complexity O(n), space O(n) (strings are immutable).

2) Using reversed() and join()
```python
def reverse_string_reversed(s: str) -> str:
    # reversed(s) returns an iterator over characters in reverse order;
    # ''.join(...) combines them into a new string
    return ''.join(reversed(s))

# Example
print(reverse_string_reversed("Hello, world!"))  # !dlrow ,olleH
```
Explanation: reversed(s) iterates characters in reverse; join builds the resulting string. Also O(n) time and O(n) space.

3) Using a manual loop (illustrative)
```python
def reverse_string_loop(s: str) -> str:
    result_chars = []
    for ch in s:
        # prepend each character (or append and reverse at end)
        result_chars.insert(0, ch)  # less efficient due to repeated inserts
    return ''.join(result_chars)

# Better manual approach: append and then reverse the list
def reverse_string_loop_efficient(s: str) -> str:
    result_chars = []
    for ch in s:
        result_chars.append(ch)
    result_chars.reverse()
    return ''.join(result_chars)

# Example
print(reverse_string_loop_efficient("Hello, world!"))  # !dlrow ,olleH
```
Explanation: The efficient manual approach appends characters to a list and then reverses the list; this avoids repeated O(n) insertions. Still overall O(n) time.

Pick the slicing approach for simplicity and readability in typical Python code.
